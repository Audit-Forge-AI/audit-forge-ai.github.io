// About

We audit websites — automatically.

AuditForge AI is a free, open website auditing platform. It crawls any public website and scores it across six categories: SEO, Accessibility, Performance, Security, Content Quality, and AI Visibility. Every finding links to a fix. No paywall, no account required.

// How it works

How it works

Paste a URL

No accounts, no installs. Drop a domain into the crawler and it walks the site automatically — discovering pages and assets without any setup.

Get a verdict

Every page is scored across six audit engines: SEO, Accessibility, Performance, Security, Content Quality, and AI Visibility. Each engine produces a weighted score tied to a public standard.

Ship a fix

Each issue ships with a concrete, prioritised recommendation mapped to the relevant standard — not a vague warning or an error code.

Built in the open

AuditForge AI is an independent project. The audit methodology, scoring weights, and referenced standards are documented publicly so any score is reproducible and auditable.

// Audit Coverage

What the platform audits

SEO

Search visibility

AuditForge AI checks every factor that affects how search engines rank a page. This includes title tags, meta descriptions, canonical URLs, heading hierarchy, internal linking, and keyword relevance.

All checks follow Google Search Central guidelines.

  • Heading structure and internal link graph
  • Missing or duplicate meta elements
  • Canonical and robots directives
AI Visibility

AI citation readiness

AI citation readiness measures how well a page is structured for large language models and AI-powered search engines to understand, extract, and cite its content.

Pages that score well here are more likely to appear in AI Overviews and answer engines.

  • Structured data and entity signal checks
  • Definition-style paragraphs and FAQ coverage
  • Unambiguous authorship signals
Accessibility

Inclusive usability

Accessibility checks test whether the site works for people with visual, motor, auditory, or cognitive disabilities.

Checks are validated against W3C WCAG 2.2 success criteria.

  • Colour contrast ratios
  • Image alternative text
  • Keyboard navigation, ARIA roles, form labels
Structured Data

Schema markup

Structured data checks detect missing, invalid, or incomplete JSON-LD that prevents search engines and AI systems from understanding what each page is about.

Validation follows Schema.org type and property definitions.

  • Type and property validation
  • Appropriate schema recommendations per page type
  • Malformed or incomplete markup flags
Performance

Core Web Vitals

Performance is benchmarked using Google's Core Web Vitals: Largest Contentful Paint (LCP), Interaction to Next Paint (INP), and Cumulative Layout Shift (CLS).

Thresholds are sourced from Google PageSpeed Insights.

  • LCP, INP, and CLS thresholds
  • Real user experience measurement
  • Actionable fixes per failing metric
// Methodology

Audit methodology

Crawl & parse

The crawler visits every publicly accessible page of a submitted domain. It parses HTML, evaluates assets, and runs checks across six audit categories.

Findings are weighted by impact and aggregated into a site-level score. Each finding links to the exact page element and maps to a specific remediation step.

External standards

Every score reflects an externally published standard. SEO checks follow Google Search Central documentation. Accessibility checks follow W3C WCAG 2.2. Performance checks use Google PageSpeed Insights thresholds. Structured data validation follows Schema.org type definitions.

No score is an arbitrary internal metric.

Content quality signals

Content quality is assessed by measuring word count, reading level, keyword concentration, and heading structure.

The presence of answer-ready content — definitions and FAQ blocks — is also scored. These signals affect both traditional search rankings and AI citation likelihood.

// Standards We Follow

Standards and references

Every AuditForge AI check is mapped to a published, externally maintained standard. These are the primary references the platform audits against.

SEO

Google Search Central

Google's official documentation covering how Search crawls, indexes, and ranks content.

AuditForge AI's SEO checks — including title tags, meta descriptions, canonical URLs, and heading structure — are validated against these guidelines.

Accessibility

W3C WCAG 2.2

The Web Content Accessibility Guidelines, version 2.2, published by the World Wide Web Consortium (W3C).

WCAG 2.2 defines success criteria for colour contrast, alternative text, keyboard navigation, ARIA roles, and form labels. AuditForge AI tests against these criteria on every crawled page.

Performance

Google PageSpeed Insights

Google's tool for measuring Core Web Vitals: Largest Contentful Paint (LCP), Interaction to Next Paint (INP), and Cumulative Layout Shift (CLS).

AuditForge AI benchmarks page performance against the thresholds published in the Web Vitals documentation.

Structured Data

Schema.org

The shared vocabulary for structured data markup, maintained collaboratively by Google, Bing, Yahoo, and Yandex.

AuditForge AI validates JSON-LD against Schema.org type definitions and flags missing or incorrectly typed properties that reduce eligibility for rich results.

// FAQ

Frequently asked questions

Who created AuditForge AI?

AuditForge AI was created by Sourav Bagchi as an independent open project. It started as a personal tool to diagnose why websites were underperforming in search and evolved into a full auditing platform covering SEO, accessibility, performance, security, content quality, and AI visibility.

Why was AuditForge AI built?

Most professional audit tools are expensive, opaque about their methodology, or require agency contracts to access. The platform was built to fill that gap — providing the same depth of analysis available in enterprise crawlers, presented transparently and without a paywall. A secondary motivation was tracking the emerging importance of AI visibility as AI-powered search engines begin to reshape organic traffic.

Why is it free?

The platform is free because the goal is access, not revenue. Small businesses, independent creators, and developers building side projects should not have to pay hundreds of dollars a month to find out if their site has a broken heading hierarchy or missing structured data. While the tool remains in beta, all audits are unlimited and free.

Is AuditForge AI open source?

AuditForge AI is built in the open as an independent project with a focus on transparency. The audit methodology, scoring weights, and the standards each check references are documented so users understand exactly what a score means and why a finding was flagged — rather than receiving a black-box number with no explanation.

How are audit scores calculated?

Each page is evaluated across six audit categories. Within each category, individual checks are weighted by their potential impact on search visibility, user experience, or AI citation likelihood. Weighted findings are aggregated into a category score, and category scores are combined into a site-level score. Every finding links to the specific page element and the external standard it violates, so there are no arbitrary deductions — every point lost traces to a documented rule.

Which standards and guidelines does the platform follow?

Audit checks are mapped to published, externally maintained standards rather than internal rules:

  • SEO — Google Search Central documentation and best practices
  • Accessibility — W3C WCAG 2.2 success criteria
  • Performance — Core Web Vitals thresholds from Google PageSpeed Insights
  • Structured data — Schema types and properties defined at Schema.org

This means every score is reproducible and auditable by the user against the same public sources.

Will AuditForge AI remain free after beta?

The intention is to keep a meaningful free tier permanently. Core auditing — SEO, accessibility, performance, and structured data checks — will remain free. If paid features are introduced in future, they will extend the platform rather than gate what is currently available. No decision on pricing has been made, and no timeline for leaving beta has been set.

How can I contribute feedback or report an issue?

Feedback is the most valuable input the project receives during beta. If an audit produces an incorrect finding, misses a real issue, or a score seems miscalibrated, reporting it directly improves accuracy for everyone. The best way to reach the team is through the contact page. Bug reports, methodology questions, and feature suggestions are all welcome.
