// Concepts

Understanding Search Visibility and AI Search

Search Visibility

Search engine visibility is a website's ability to appear in relevant results. It depends on technical health, content relevance, backlinks, and user experience signals working together.

AI Search

AI search tools answer questions directly, pulling facts from a small number of sources. To get cited, a page needs clear structure, direct definitions, and accurate facts. Vague or buried information rarely gets quoted.

// Maintenance

Website Health, Best Practices, and Common Mistakes

Website Health Monitoring

Website health changes constantly. New content, code changes, and third-party scripts can introduce issues overnight. Ongoing monitoring catches regressions before they affect traffic.

SEO Best Practices

  • Write a unique, descriptive title tag for every page
  • Use one H1 per page and a logical heading order
  • Compress images and serve modern formats
  • Fix broken links before they accumulate
  • Add structured data to help search engines and AI systems understand content

Common SEO Mistakes

  • Duplicate or missing meta descriptions
  • Skipping heading levels, such as jumping from H1 to H3
  • Ignoring mobile usability
  • Treating SEO as a one-time task instead of an ongoing process
  • Publishing thin content with little real value

// Definitions

Key SEO and AI Visibility Definitions

SEO Audit
A review of a website's technical health, content, and visibility that identifies issues limiting search performance.
Technical SEO
The practice of improving how search engines crawl, render, and index a website — covering speed, structure, and indexability.
Structured Data
A standardized format that helps search engines understand the meaning of page content beyond plain text.
Schema Markup
Code that adds structured data to a webpage using a shared vocabulary that search engines recognize.
Core Web Vitals
Performance metrics that measure loading speed, interactivity, and visual stability from a real user's perspective.
Accessibility
The practice of designing websites that people with disabilities can use, including those relying on screen readers or keyboards.
AI Visibility
The ability of content to be found, understood, and cited by AI-powered search systems and assistants.
Search Engine Visibility
A website's ability to appear in relevant search results for the queries it targets.
Entity
A distinct, well-defined thing — a person, place, organization, or concept — that search engines can recognize and connect across the web. Strong entity signals help AI systems understand who or what a page is about.
Search Intent
The underlying goal behind a search query. Pages that match search intent — informational, navigational, or transactional — tend to rank and perform better.
Canonical URL
The preferred version of a webpage, declared using a rel="canonical" tag. It tells search engines which URL to index when duplicate or similar content exists across multiple addresses.
Indexability
Whether a search engine can crawl and add a page to its index. Pages blocked by robots.txt, noindex tags, or crawl errors cannot appear in search results.
Robots.txt
A file at the root of a website that tells search engine crawlers which pages or sections they are allowed to access. Misconfigured robots.txt files can accidentally block important content from being indexed.
